Metodologia Tradicional Microsoft Solutions Framework (MSF).


HISTORIA

MSF fue presentado por primera vez por Microsoft como la versión 1.0 en 1993, y una versión 2.0 fue lanzada en 1997. En 2002, se lanzó la versión 3.0 de MSF. Modificó la versión 2.0 de las siguientes maneras: Modelos combinados previamente separados en modelos unificados de Equipo y Proceso diseñados para la aplicación en una variedad de tipos de proyectos que incluyen implementación, integración de software empresarial y proyectos de desarrollo. Dobló los modelos de Desarrollo de aplicaciones e Implementación de infraestructura en un solo Modelo de proceso que consta de cinco fases. Se agregaron disciplinas de gestión de proyectos y gestión de preparación. Se hicieron cambios a la Disciplina de Gestión de Riesgos. Se agregaron enlaces entre MSF y Microsoft Operations Framework (MOF). Se agregó un programa para profesionales de MSF diseñado para capacitar a las personas para liderar o participar en proyectos de MSF. MSF versión 4.0 se lanzó en 2005. El lanzamiento fue una actualización importante del Modelo de proceso (ahora llamado Modelo de gobierno) y el Modelo de equipo. [2] MSF 4.0 incluyó técnicas para dos metodologías separadas: MSF para el desarrollo de software ágil (MSF Agile) y MSF para la mejora de procesos CMMI (MSF4CMMI).
