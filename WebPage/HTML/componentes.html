Metodologia Tradicional Microsoft Solutions Framework (MSF).


COMPONENTES

MSF 4.0 es una combinación de un metamodelo que puede usarse como base para procesos de ingeniería de software prescriptivos, y dos procesos de ingeniería de software personalizables y escalables. El metamodelo de MSF consta de principios fundamentales, un modelo de equipo y ciclos e iteraciones. MSF 4.0 proporciona un marco de orientación y principios de nivel superior que se puede asignar a una variedad de plantillas de proceso prescriptivas. Está estructurado en metodologías descriptivas y prescriptivas. La componente descriptiva se llama el MSF 4.0 metamodelo, que es una descripción teórica de las SDLC mejores prácticas para crear metodologías SDLC. Microsoft opina que las organizaciones tienen dinámicas divergentes y prioridades contrarias durante su desarrollo de software. Algunas organizaciones necesitan un entorno de desarrollo de software sensible y adaptable, mientras que otras necesitan un entorno estandarizado, repetible y más controlado. Para satisfacer estas necesidades, Microsoft representa el metamodelo de MSF 4.0 en dos plantillas metodológicas prescriptivas que proporcionan una guía específica del proceso, para el desarrollo de software ágil (MSF4ASD) y para el Modelo de madurez de capacidad (MSF4CMMI). Estos procesos de ingeniería de software se pueden modificar y personalizar según las preferencias de la organización, el cliente y el equipo del proyecto. La filosofía de MSF sostiene que no existe una estructura o proceso único que se aplique de manera óptima a los requisitos y entornos para todo tipo de proyectos. Por lo tanto, MSF admite múltiples enfoques de proceso, por lo que puede adaptarse para respaldar cualquier proyecto, independientemente de su tamaño o complejidad. Esta flexibilidad significa que puede soportar un amplio grado de variación en la implementación de procesos de ingeniería de software al tiempo que conserva un conjunto de principios y mentalidades centrales.
